# Модель обнаружения аномалий в числовых последовательностях

Учебный MVP-проект по кейсу **«Модель для обнаружения аномалий в числовых последовательностях»**.

Приложение генерирует или принимает числовой временной ряд, находит аномальные точки несколькими базовыми методами и показывает результат на интерактивном графике.

## Что решает проект

Аномалии — это резкие отклонения от нормального поведения данных. Такие отклонения могут означать сбой датчика, подозрительную транзакцию, всплеск нагрузки или ошибку в данных. Проект демонстрирует базовый пайплайн: генерация данных → алгоритм → визуализация → отчет.

## Возможности

- генерация синтетического временного ряда с трендом, сезонностью, шумом и выбросами;
- загрузка пользовательского CSV-файла;
- три метода поиска аномалий:
  - Z-score;
  - IQR;
  - Isolation Forest;
- интерактивный график с выделением найденных точек;
- таблица результатов со score и флагом `is_anomaly`;
- базовые метрики: количество точек, число аномалий, доля аномалий, среднее и стандартное отклонение;
- скачивание результата в CSV.

## Формат входного CSV

Обязательная колонка:

| Колонка | Описание |
|---|---|
| `value` | числовое значение ряда |

Опциональная колонка:

| Колонка | Описание |
|---|---|
| `timestamp` | дата/время или индекс точки |

Пример:

```csv
timestamp,value
2026-01-01,50.2
2026-01-02,52.1
2026-01-03,92.4
```

## Как работают методы

### Z-score

Считает, насколько далеко точка находится от среднего значения в единицах стандартного отклонения. Если абсолютное значение Z-score выше порога, точка считается аномалией.

### IQR

Использует межквартильный размах. Точки ниже `Q1 - k*IQR` или выше `Q3 + k*IQR` считаются выбросами.

### Isolation Forest

Готовый ML-алгоритм из `scikit-learn`, который изолирует редкие и необычные наблюдения. В MVP используется как дополнительный метод сравнения с простыми статистическими подходами.

## MVP-ограничения

- данные синтетические и используются для демонстрации;
- методы не обучаются под конкретную предметную область;
- для production-версии стоит добавить мониторинг потоковых данных, хранение истории, настройку порогов по типам сигналов и оценку качества по размеченным данным.
